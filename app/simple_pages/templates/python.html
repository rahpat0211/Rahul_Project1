On the left side you will see text which identifies and explains what the page will be about.

Flask Blueprint Documentation

Flask Blueprint Documentation helps you organize your files and work within the application making your code more maintainable and easier to debug. As shown in the image above, the first two lines are importing the libraries to access Flask Blueprints. Line 4 describes what the Blueprints name is (simple_pages), where the files for the web pages will be (templates), and where the static files such as JS,CSS, Images will be (static). Line 8-9 makes the bundle of controllers that make the correct routing between pages located in (templates). Finally, the show function renders the template based upon the page names located in (templates), if no pages are found that work with the function, it will throw an exception error.

Outline


  • Rahul_Project1 <- Root of the project
    • docker-compose.yml <- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku
    • Dockerfile <- File used to create an image to run in a container that runs the program
    • readme.md <- File used to describe the project
      • .github/workflows <- Sub directory of Rahul_Project1 that is used for development and production workflows.
        • dev.yml <- Development workflow configuration. Used to send updated and testing files.
        • prod.yml <- Production workflow configuration. Used as the main configuration file for the latest updates.
      • .app <- Sub directory of Rahul_Project1 that is used for web pages.
        • static <- Sub directory of .app that holds CSS files, JS files, and images.
          • images <- Sub directory of .app that holds image files.
          • js <- Sub directory of .app that holds JS files.
          • css <- Subdirectory of .app that holds CSS files
            • style.css <- CSS file for index.html
            • cicd.css <- CSS file for cicd.html
            • docker.css <- CSS file for docker.html
            • git.css <- CSS file for git.html
            • python.css <- CSS file for python.html
        • templates <- Sub directory of .app that holds base.html
          • base.html <- HTML file to create the homepage
        • simple_pages <- Sub directory of .app that holds the web pages and python script to run index page.
          • templates <- Sub directory of simple_pages that holds all the web pages
            • index.html <- Index Page
            • cicd.html <- Continous Integration & Deployment Page
            • docker.html <- Docker Page
            • git.html <- Git Page
            • python.html <- Python/Flask Page
          • _init_.py <- Python script that holds flask blueprint documentation.
