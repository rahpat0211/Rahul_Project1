How to review code using Development Server

To start off, once you have made the changes you want to commit and pushed your branch, navigate to Github and go to your repository. Next navigate to Pull Request and click on New Pull Request which should be located on the right side of the page. Here click on the dropdown menu of compare:master and select your branch where you made the commits on. Here you will be able to review your code (Red is the original code)(Green is your edited code) before making a merge pull request. Next go on your Heroku account and click on your development server app. This will show you what the web pages look like when you pushed your branch on pycharm.

How to deploy master to production server

To start off, you will need to update the configuration file, prod.yml which is located in workflows directory. From there, you need to make sure that on line 6 it says master since we will be deploying from this branch. Next, on line 58, we will need your Heroku app name which in this case is production. Next, on line 59, we will need your Heroku email address. Once this is all done go ahead and save your file. Finally, go on your GitHub and create secret keys for HEROKU_API_KEY and enter your API key. After finishing that, go back onto pycharm, navigate to Git and click on Commit. Here you can update your commit with what you want to say, after click Push. This will allow you to deploy your master branch to your production server.

How to deploy image of your project to Docker Hub, when master is updated

To start off, you will need to update the configuration file, prod.yml which is located in workflows directory. From there, you need to navigate to line 42 and enter your Dockerhub repository name (EX <- rup1141/patel-r-project1:master). Next you will need to navigate to your github account and create secrets keys for DOCKER_USERNAME & DOCKER_PASSWORD. After finishing that, go back onto pycharm, navigate to Git and click on Commit. Here you can update your commit with what you want to say, after click Push. This will allow you to deploy an image of your project to Dockerhub when your master is updated.
