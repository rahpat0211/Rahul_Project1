Encapsulation:

Reference

This is the packing of data and methods into a class so that you can hide the information and restrict the access from outside.

Inheritance:

Reference

This refers to defining a new class with little or no modification to an existing class.

Usually when you create a new class it will be called the child class and the class where this inherits it from will be called the parent class. If an attribute is not found in the child class itself, it will continue to search to itâ€™s parent class.

As shown in the images above, Addition(), Subtraction(), Multiplication(), and Division() are all subclasses of Calculation().

These subclasses inherits 2 abstract methods, create and convert_args_to_tuple_of_float and 1 method, __init__. Inheriting these methods means that we can use them as if they were defined in the Addition(), Subtraction(), Multiplication(), and Division() classes.

When you create an instance for Addition(), Subtraction(), Multiplication(), or Division(), the __init__ function will also convert the tuple of random data types to the data type float, in which it can be used as a list and be able to edit the calculation.

Polymorphism:

Reference

This refers to the ability of the function with the same name to carry different functionality altogether.

Abstraction:

Reference

This refers to hiding the internal implementation of a process or method from the user. This is how the user knows what he/she is doing but not how the work is being done.

A


In order to declare an Abstract class, you will first need to import the calculator.operations module as shown in above image on line 2.

The class Calculator shown in the Image(A) above, is the abstract class that inherits from the Addition(), Subtraction(), Multiplication(), and Division() classes from the calculator.operations module. It contains an abstract method create() and convert_args_to_tuple_of_float() which are visible by the user.

After the user creates objects and invokes the abstract methods, the hidden definitions for the abstract methods will run. These definitions are hidden from the user and will only be invoked on the Classes:

  • Addition()
  • Subtraction()
  • Multiplication()
  • Division()

This is because these classes are an extension of the abstract method and are not classified as an abstract method.
